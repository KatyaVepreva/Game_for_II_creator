# Бриф за 60 секунд

Интерактивная мини-игра для браузера про то, как хороший бриф спасает проект, а размытые формулировки растят хаос.

Стек: `React + TypeScript + Tailwind CSS` (без backend, без БД, без авторизации).

## Что внутри

- 7 вопросов с таймером 60 секунд.
- 2 шкалы: ясность и хаос.
- Реакции после каждого ответа (в том числе вариативные по типу ответа).
- Персонаж-помощник Луна с подсказками.
- Финальный результат с категорией и рекомендацией.
- Мини-разбор: сильные стороны и что уточнить.
- Кнопки копирования:
  - итогового результата;
  - вирального вызова для подруги/коллеги;
  - шаблона "идеального брифа".
- UX-полировка:
  - горячие клавиши `1-4`;
  - индикатор оставшихся вопросов;
  - анимационный отклик шкал.
- Локальный рекорд в `localStorage` (виден на стартовом и финальном экранах).

## Запуск

```bash
npm install
npm run dev
```

После запуска открой адрес из терминала (обычно `http://localhost:5173`).

## Production-сборка

```bash
npm run build
npm run preview
```

## Структура проекта

- `src/App.tsx` — центральная логика игры, подсчет, copy actions, localStorage.
- `src/data/questions.ts` — вопросы, варианты, очки, базовые и вариативные реакции.
- `src/components/StartScreen.tsx` — стартовый экран + блок рекорда.
- `src/components/GameScreen.tsx` — игровой экран, таймер, ответы, хоткеи.
- `src/components/ProgressBars.tsx` — шкалы ясности/хаоса.
- `src/components/LunaMessage.tsx` — карточка Луны.
- `src/components/ResultScreen.tsx` — финал, мини-разбор, кнопки копирования.
- `src/index.css` — глобальные стили и анимации.

## Идея игры

Игрок собирает бриф для нейрофотосессии/AI-визуала.  
Чем конкретнее ответы — тем выше ясность.  
Чем больше "сделайте красиво" и "хочу вау без примеров" — тем выше хаос.

Публикация настроена через GitHub Pages и GitHub Actions.
